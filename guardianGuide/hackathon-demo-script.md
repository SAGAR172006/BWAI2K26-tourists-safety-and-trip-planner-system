- Show Strike 1 warning

- Hover a red zone — show popup with score
- Show Must Visit + Must Avoid lists populating from zones

### Step 6: SOS (20 sec)
- Navigate to SOS screen
- "Completely different theme — high contrast red for emergencies"
- Show emergency directory

## Judge Questions — Prepared Answers

**Q: How accurate are the safety zones?**
"We use Ragas evaluation to measure faithfulness and context precision. Our target is 95%+ accuracy. Sentiment scores are cached for 24 hours and recalculated when new data is available."

**Q: How do you handle the budget guardrail?**
"The Two-Strike rule: first offense is a warning, second offense blocks the API call entirely. This saves compute cost AND prevents users from planning trips they literally cannot afford."

**Q: Is user data safe?**
"Yes. Supabase Row Level Security ensures users can only access their own data at the database level — not just in code. PII is stripped before any LLM call. Gmail access is read-only and can be revoked at any time."

**Q: How does it scale?**
"Frontend on Vercel (auto-scaling). Backend on Google Cloud Run (scales to zero, pay per request). Redis caching reduces AI inference calls by ~80% for repeat destinations."
